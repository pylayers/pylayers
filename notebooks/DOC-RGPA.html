RGPA¶

TOA¶

At the begining it is require to create the Constraint Layer Array (CLA) which gathers all the geometric constraints of the considered scenario.

C=CLA()

Instanciate TOA objects. Notice that the ids have to be different.

T1=TOA(id=0,value = toa1, std = 1.0, p = ptoa1)
T2=TOA(id=1,value = toa2, std = 1.0, p = ptoa2)
T3=TOA(id=2,value = toa3, std = 1.0, p = ptoa3)
T4=TOA(id=3,value = toa4, std = 1.0, p = ptoa4)

Then the TOA constrains are added one after an other to the CLA list.

C.append(T1)
C.append(T2)
C.append(T3)
C.append(T4)

List all constraints of the CLA

Get informations from the CLA :

  • type : TOA / RSS
  • p : Position of the origin of the constraint
  • value : power ( RSS ) / time in ns ( TOA)
  • std : standard deviation of value
  • runable : does the constraint has a position p ?
  • obsolete : does the value has been obtained recently ?
  • usuable : runbale AND NOT obsolete
  • evlauated : obsolete

RSS¶

The RSS is a quantity which is weakly related to distance. To model the RSS/distance dependenct the Path Loss Shadowing model is widely used. To define a path loss shadowing model use the dedicated class PLSModel as follow

M=PLSmodel(f=3.0,rssnp=2.64,d0=1.0,sigrss=3.0,method='mode')

For simulation purpose it is possible to get a realization of an RSS value from a given distance.

toa1

PLmean = M.getPLmean(toa1)
